GITHUB COMMANDS

S.No. Command Description
1 Git status : It is used to check the status of the git repositery
2 git init : It is used to initialize the git repositery and it makes a ".git" folder.
3 git config --global user.name "" : This command inculudes the configuration of the user's username with a global scope.
4 git config --global user.email "" : This command includes the configuration of the user's email with global scope.
5 git add ./ : This command is used to stage the repositery to the git. And the '.' is used for selecting all the files.
6 git commit -m "" : This command is used to commit or save the changes with an updated message. Each commit generates a unique Hash code.
7 git remote add origin : It will add the remote named as origin. Origin will contain the link and we can call the link by origin.
8 git push -u origin master : This command is used to push or send the changes in the repositery to the github. It will push the master branch to the remote repositery or the Github.
9 git log : It is used to check the logs or activities done by the users.
10 git reset ./ : It is used to reset or unstage the repositery .
11 git checkout ./ : It is used to delete the new or unwanted changes in the mentioned file.
12 git diff : It is used to find the difference of the updated file and old file. But the file should not be staged because it compaeres the updated file with the unstaged file.
13 git .gitignore : It is used to ignore some particular files which we dont want to push on the github. It will ignore them while staging. It create a file then you need to add some file names in .gitignore file. A slash will be used if we want .gitignore to ignore a file in any directory.
14 git branch : It is used to list all the branches.
15 git branch "new branch name" : It will create a new branch.
16 git checkout "branch name" : It will switch us to the named branch.
17 git merge " branch name" : It will merge the mentioned branch with current branch.
18 git checkout -b "new branch name" : It will create a branch and get switched to new branch simultaneously.
19 git remote : It will show the remote origin.
20 git remote -v : It will show the URLs which are showing that from where are you going to push or pull.
